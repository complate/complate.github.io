We have prepared some sample projects in order to get you started.

But first of all, here’s how all those puzzle pieces that complate consists of, belong together:
complate is – in most cases – a combination of the core-library and a language-specific adaptor to its target environment.

complate-core

The core library is the same across all possible runtime environments, which allows us to share our components with different systems, even if those are not based on the same technology.

The core library itself consists basically of two parts: complate-stream and complate-dom.

complate-dom may also be used if you want to render complate components directly in the browser. In this case, no adaptor is needed.

complate-adaptors

Then there are several adaptors, which enable rendering of complate components within a specific technology.
