complate

rendering HTML with expressive, declarative and composable abstractions (i.e. components) – independent of any particular technology choice

say what you mean

— Web Components: The Long Game

<Card title="Hello World">
    <p>lorem ipsum dolor sit amet</p>
    <Button label="toggle" />
</Card>

turns into

<article class="card">
    <h3 class="card-title">Hello World</h3>

    <div class="card-body">
        <p>lorem ipsum dolor sit amet</p>
    </div>

    <div class="card-controls">
        <button type="button">toggle</button>
    <div>
</article>

complate leverages JSX to convert such markup abstractions (“macros”), optionally interspersed with JavaScript expressions, into HTML – either on the server or on the client side.

Because the underlying library consists only of a few JavaScript functions, complate macros can be used across a variety of tech stacks:

  • Express (Node.js)
  • Ruby on Rails (Ruby)
  • Spring MVC (Java)
  • etc.

It’s also often used for pattern libraries or as a static-site generator.
